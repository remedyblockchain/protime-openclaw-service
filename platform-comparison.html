Configuration risks — platform-independent

API keys in tools.md or committed .env files

Credentials stored in any file tracked by git are effectively public. Scrapers harvest exposed keys within minutes of a push — and git history preserves them even after deletion. Secrets belong in a .env explicitly excluded from version control, rotated immediately if exposure is suspected.

No backup repository = no disaster recovery

Agent personas, routing logic, prompt templates, and tool configs that exist only on a single device have no recovery path. One failed NVMe, one corrupted config — and months of work are gone. Every OpenClaw configuration must live in a private, regularly-pushed git repository.

The model matters as much as the hardware

Right model. Right task. Zero waste.

Not every agent task needs a frontier model. Protime maps each task in your system to the most cost-effective model that can handle it — local where it makes sense, cloud where it has to be. The result: a system that performs like a premium setup and costs like a lean one.
